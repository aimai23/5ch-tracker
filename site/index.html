      ARAKAWA BRIEF 使い方

      ARAKAWA BRIEF は 5ch / Reddit の話題を監視し、短期の温度感と監視優先度を整理するための画面です。売買推奨ではなく、観測・警戒のためのダッシュボードとして使ってください。

      1. ARAKAWA BRIEF の基本

      • SWING / LONG: 監視の時間軸を切り替えます。数日〜数週を SWING、数週〜数か月を LONG として扱います。
      • Watchlist: 話題量・継続性・リスク情報をまとめた監視対象です。投資助言ではありません。
      • Changes: 直近スナップショット比で注目が増減した銘柄を優先表示します。
      • Cautions / Calendar: ボラティリティ要因とイベント日程の整理です。必ず一次情報で再確認してください。

      2. 画面の読み方（実務）

      • Headline: まず相場全体の温度感を確認します。
      • Watchlist: 監視優先度の高い銘柄を把握します。
      • Changes: 前回比で監視優先度が動いた銘柄を追います。
      • Cautions / Calendar: 近いリスクイベントを先に確認します。

      3. 運用ルール

      • ヘッドラインで地合い確認 → Watchlist で候補抽出 → Changes で優先順位更新、の順で使います。
      • ONGENBURG OMEN の詳細ルールは MARKET タブ下部の「判定ルール」から確認してください。
      • イベント日（Calendar）前後は、通常よりポジションサイズを抑える運用を推奨します。

      4. 注意点

      • 話題量は価格方向を保証しません。価格・出来高・決算情報を必ず照合してください。
      • 速報値は改定される場合があります。重要判断は直前データで再確認してください。
      • 本画面は情報整理用途です。最終判断は必ずご自身で行ってください。

      ※ 本ブリーフは情報整理を目的とした表示であり、投資助言ではありません。最終判断は必ずご自身で行ってください。

      ONGENBURG OMEN 判定ルール

      本表示はヒンデンブルグ系シグナルを、ルールを固定して一貫運用するための監視UIです。流派差が大きい指標のため、下記ルールを明示して点灯判定しています。

      1. 採用データ

      NYSE breadth: WSJ Market Diary（Advances / Declines / New Highs / New Lows）

      トレンド参照: Yahoo Finance（^NYA 終値）

      2. ベース条件（点灯候補）

      同時成立: 新高値・新安値がどちらも閾値以上

      偏り制限: 新高値 <= 新安値 x 2

      閾値: ceil(2.8% x (Advances + Declines))。欠損時のみ Issues Traded を代替

      3. Strict 条件（TRIGGER 判定）

      クラスタ: 30 営業日内にベース条件が 2 回以上

      トレンド: ^NYA が 50 営業日前終値を上回る

      内部モメンタム: McClellan Oscillator（EMA19-EMA39 of Net Advances）< 0

      履歴条件: strict 判定は履歴 40 営業日以上で有効化

      4. UI 表示ルール

      • LAMP ON: WATCH / TRIGGER 系の状態。
      • 点灯回数: TOTAL / 30D / 90D を表示。
      • 履歴: 点灯日だけを表示（最新順）。

      5. 運用上の注意

      • ヒンデンブルグオーメンは流派差があります。本UIは上記ルールで固定運用します。
      • 単独指標で将来を断定せず、VIX・クレジット・マクロ指標と併用してください。
      • 速報値改定や欠損補完により、過去履歴が更新される場合があります。
